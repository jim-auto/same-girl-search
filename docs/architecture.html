Architecture Notes

小さく始めて、壊さず広げる

MVP は、画像フォルダから embedding を作り、SQLite と FAISS に保存し、FastAPI と Next.js UI から検索する直接的な構成です。

Module Ownership

  • libs/embedding: model loading, face detection, normalized embedding output
  • libs/vector_search: FAISS persistence and ID mapping
  • apps/api: HTTP, SQLite persistence, ingestion orchestration, deletion
  • apps/web: query upload and result rendering
  • libs/crawler: lawful crawler policy, robots checks, future source adapters

Index Contract

  1. SQLite images.id is the canonical ID.
  2. FAISS row IDs map to SQLite IDs through a JSON sidecar file.
  3. Embeddings are normalized before insertion, so FAISS inner product equals cosine similarity.

Rebuild Strategy

If a record is deleted or the FAISS sidecar is missing, rebuild FAISS from SQLite embeddings. This keeps deletion simple and avoids early over-abstraction.
